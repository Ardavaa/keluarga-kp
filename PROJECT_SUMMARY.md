# Project Summary: Lecturer Research & Collaboration Dashboard (`keluarga-kp`)

This document provides a detailed overview of the **"Lecturer Research Topic Visualization and Expertise Mapping Dashboard to Support AI Collaborations in FIF"** project (repository: `keluarga-kp`). It is built as a web-based presentation layer for the Faculty of Informatics (FIF) at Telkom University.

---

## 1. Project Overview & Background

The `keluarga-kp` project is a Laravel-based web dashboard designed for the **FIF AI Task Force (Satgas AI FIF)** at Telkom University. Its primary goal is to map research capabilities, identify dominant academic topics, cluster lecturer expertise, and recommend co-authorship/collaboration opportunities among faculty members.

*   **Project Name:** `keluarga-kp`

## 2. System Architecture & Data Flow

The overall system is divided into two distinct components: the **Data Pipeline (Scraper & AI Processor)** and the **Web Application (Dashboard)**.

```mermaid
flowchart TD
    %% Data Sources
    subgraph Data Sources
        SINTA[SINTA Portal]
        OpenAlex[OpenAlex API]
        Excel[Keilmuan Dosen FIF.xlsx]
    end

    %% Python Pipeline
    subgraph Data Pipeline (Python Repository)
        Scraper[Playwright Scraper]
        GeminiParser[Gemini AI Parser]
        Recommender[Vector Similarity Recommender]
    end

    %% Database
    subgraph Database (PostgreSQL)
        DB[(PostgreSQL Database)]
    end

    %% Laravel App
    subgraph Presentation Layer (Laravel keluarga-kp)
        Laravel[Laravel 13.x Backend]
        Blade[Blade Template Engine]
        Vite[Vite + Tailwind v4]
        ChartJS[Chart.js / vis-network]
        Exports[Excel / PDF Exports]
    end

    %% Connections
    SINTA --> Scraper
    OpenAlex --> Scraper
    Scraper --> GeminiParser
    Excel -->|Manual Command / Seeder| DB
    GeminiParser -->|Profile & Citations| DB
    Recommender -->|Similarity Scores| DB
    
    DB <-->|Direct Connection - Read Mostly| Laravel
    Laravel --> Blade
    Blade --> ChartJS
    Blade --> VisNetwork[vis-network Graph]
    Blade --> Exports
```

1.  **System of Record (Python Scraper Repository):**
    *   Uses Playwright to scrape publication and citation data.
    *   Queries SINTA and OpenAlex API.
    *   Utilizes Gemini AI API to parse research keywords, abstracts, and classify AI categories.
    *   Calculates similarity embeddings using vectors and generates collaboration recommendations.
2.  **Shared Database (PostgreSQL):**
    *   A shared PostgreSQL database acts as the single source of truth. It contains precomputed tables for lecturers, publications, keywords, co-authorship frequencies, and vector similarities.
3.  **Presentation Layer (`keluarga-kp`):**
    *   A clean, responsive Laravel web app that connects directly to the PostgreSQL database, retrieves relational data, and exposes it through charts, interactive graphs, and search interfaces.

## 3. Database Schema & Data Model

The database is structured to support relational queries and mapping, matching the structure generated by the Python pipeline. The Laravel models are configured with `$timestamps = false` as the raw scraping database does not utilize Laravel's default timestamps.

### Entity-Relationship Diagram (ERD)

```mermaid
erDiagram
    LECTURERS {
        int id PK
        string name_with_title
        string code UNIQUE "NIP / Lecturer Code"
        string study_program
        string research_group "CITI / DSIS / SEAL"
        string academic_rank "JFA"
        string field "Primary keilmuan"
        string photo
        int citation_count
        int h_index
        int i10_index
        jsonb ai_categories
        jsonb sinta_metrics
    }
    
    PROFILES {
        int id PK
        int lecturer_id FK
        string platform "SINTA, Google Scholar, Scopus, ORCID"
        string url
    }

    PUBLICATIONS {
        int id PK
        int lecturer_id FK
        string title
        string journal_publisher
        int year
        int citation_count
    }

    KEYWORDS {
        int id PK
        int lecturer_id FK
        string keyword
    }

    RESEARCH_INTERESTS {
        int id PK
        int lecturer_id FK
        string interest
    }

    COAUTHORS {
        int id PK
        int lecturer_id FK
        string coauthor_name
        int collaboration_count
    }

    RECOMMENDATIONS {
        int id PK
        int lecturer_id FK
        int recommended_lecturer_id FK
        float score "Vector similarity score"
        jsonb reasons "List of common grounds"
    }

    COLLABORATIONS {
        int id PK
        int lecturer_id_1 FK "Where lecturer_id_1 < lecturer_id_2"
        int lecturer_id_2 FK
        int collaboration_count
        jsonb shared_publications "List of mutual publications"
    }

    LECTURERS ||--o{ PROFILES : "hasMany"
    LECTURERS ||--o{ PUBLICATIONS : "hasMany"
    LECTURERS ||--o{ KEYWORDS : "hasMany"
    LECTURERS ||--o{ RESEARCH_INTERESTS : "hasMany"
    LECTURERS ||--o{ COAUTHORS : "hasMany"
    LECTURERS ||--o{ RECOMMENDATIONS : "recommendationsGiven"
    LECTURERS ||--o{ RECOMMENDATIONS : "recommendationsReceived"
    LECTURERS ||--o{ COLLABORATIONS : "collaborationsAsFirst"
    LECTURERS ||--o{ COLLABORATIONS : "collaborationsAsSecond"
```

### Key Relational Implementations:
*   **Collaborations Relationship:** To avoid double-entry, pairs are stored exactly once with `lecturer_id_1 < lecturer_id_2`. The `Lecturer` model features a custom helper method `collaborations()` that merges `collaborationsAsFirst` and `collaborationsAsSecond` to list all collaborations of a lecturer regardless of key placement.
*   **Recommendations:** Relates two lecturers using vector similarity. `reasons` is cast as a JSON array listing the matching topics, fields, and common keywords.

## 5. Dashboard Features & Page Specs

### 1. Main Dashboard
*   **KPI Summary:** Four cards showcasing the total counts of Lecturers, Publications, Co-authorship Collaborations, and Unique AI Categories.
*   **Research Groups (KK):** Breakdown of lecturers grouped by CITI, DSIS, and SEAL.
*   **Latest Publications & Top Collaborations:** Polished lists containing recent research and highly frequent co-authorship pairings.
*   **Export Actions:** Allows exporting the full dashboard metrics summary into Excel or PDF.

### 2. Lecturer Directory (Search & Global Filters)
*   **Comprehensive Filter Bar:** Includes real-time search by name, and dropdown selectors for:
    *   Study Program (*Program Studi*)
    *   Research Group (*Kelompok Keahlian*: CITI/DSIS/SEAL)
    *   Core AI Field (*Keilmuan*)
    *   Publication Year
*   **Dynamic Sorting:** Sort lecturers by name, research group, and study program.
*   **Excel/PDF Export:** Allows downloading the filtered grid list of lecturers.

### 3. Lecturer Profile Details
*   **Academic Identity:** Showcases JFA rank, lecturer code (NIP), study program, and KK badge.
*   **Research Metrics:** Includes cards for SINTA citations, H-index, and i10-index.
*   **External Profiles:** Hyperlinks to Google Scholar, Scopus, SINTA, and ORCID profiles (displays greyed out if not registered).
*   **Metadata:** List of research interests, keywords, and publication history.

### 4. Expertise Map
*   **KK Tab Selector:** Dynamically filters and displays cards of lecturers according to their respective research groups (CITI, DSIS, SEAL).

### 5. Dominant Topics
*   **AI Topic Distribution:** An interactive Chart.js doughnut graph showcasing the proportion of lecturers contributing to specific AI disciplines, accompanied by an detailed tabular breakdown.

### 6. Collaboration Graph
*   **Network Visualizer:** An interactive `vis-network` canvas showing co-authorship relationships. Node sizes scale with their connection count, and colors correspond to their research groups. Includes a tabular breakdown below showing mutual publications.

### 7. Recommendations
*   **Top Matches:** Focuses on the top 3 vector similarity pairings, rendering cards detailing mutual fields and matching keywords.
*   **Ranks Table:** Displays subsequent pairings sorted in descending order of similarity score.

## 6. Design System & Visual Identity

The dashboard reflects the corporate visual identity of **Telkom University** with customized, functional dashboard adjustments:

*   **Primary Palette:**
    *   TelU Red: `#9F1521` (Used for header accents, primary buttons, and active tabs)
    *   Deep Red: `#AC000F`
    *   Dark Red: `#8F131E` (Used for hover states)
    *   TelU Navy: `#002147` (Secondary/accent color)
*   **Research Group Coding:**
    *   **CITI:** Red theme
    *   **DSIS:** Green theme
    *   **SEAL:** Blue theme
*   **Typography:** The modern sans-serif **Inter** font is self-hosted via `laravel-vite-plugin/fonts` to ensure performant, consistent offline rendering.
*   **Layout Structure:** Migrated from a top-bar layout to a sticky **Left Sidebar** layout on desktops and a slide-out drawer layout on mobile screens, providing a persistent navigation tree.
*   **De-AI-ify UI Philosophy:** Based on recent requests, the dashboard was stripped of typical "AI-generated mockup" styling cues. It relies on clean, professional, and readable interfaces:
    *   *Removed:* Gradient heroes, background glowing mesh/orbs, pulsing badge indicators, rounded gradient initial avatars, and floating hover translations (`translate-y-[-4px]` with shadow expansion).
    *   *Implemented:* Clean 1px solid borders, flat card structures, standard layout dividers, and plain table records.

## 8. Setup & Development Guide

Follow these steps to run the project locally.

### Prerequisites
*   PHP **8.3+** with extension: `pdo_pgsql`, `pgsql`, `mbstring`, `openssl`, `curl`, `fileinfo`, `zip`, `gd`, `intl`
*   Composer **2.x**
*   Node.js **LTS** & npm

### Installation
1.  **Clone the repository:**
    ```bash
    git clone https://github.com/Ardavaa/keluarga-kp.git
    cd keluarga-kp
    ```
2.  **Install dependencies:**
    ```bash
    composer install
    npm install
    ```
3.  **Environment Setup:**
    ```bash
    cp .env.example .env
    php artisan key:generate
    ```
    *Modify `.env` to configure your PostgreSQL or SQLite database.*
4.  **Database Migration & Seeding:**
    ```bash
    php artisan migrate --seed
    ```
5.  **Run Development Server:**
    ```bash
    # Terminal 1: Vite asset compiler
    npm run dev

    # Terminal 2: Laravel server
    php artisan serve
    ```
6.  **Optional: Import Excel Roster:**
    If you have the Excel spreadsheet, you can import the lecturer registry using the custom command:
    ```bash
    php artisan import:lecturers "/path/to/Keilmuan Dosen FIF.xlsx"
    ```

Open your browser and navigate to `http://127.0.0.1:8000` to preview the dashboard.
